개요

블로그 템플릿화 작업의 첫 단계는 기존 레포에서 템플릿 코드를 분리하는 것이었습니다. 어떻게 템플릿 코드를 분리하는 과정을 거쳤는지에 대해 알아보겠습니다.

템플릿 코드 분리 과정

  1. 레포 분리 준비

    먼저, 기존의 통합 개발 레포에서 템플릿 코드를 분리하기 위한 준비 작업을 진행했습니다. 이 과정에서는 두 개의 레포가 존재했습니다:

    • 초기 개발/테스트용 레포: 초기 개발 단계에서 프로토타이핑과 테스트를 위해 사용되던 레포입니다.

    • 통합 개발 레포: 실제 운영 중인 블로그로, 포스트와 템플릿 코드가 함께 관리되고 있었습니다.

    통합 개발 레포는 초기 개발/테스트용 레포의 개발 내용을 기반으로 구축되었으며, 이후 사용자의 포스팅 내용 추가와 몇몇 기능 업데이트가 이루어졌습니다.

    초기 개발/테스트용 레포는 테스트를 위한 contents 파일들을 제외하면 포스트나 배포용 빌드 파일에 대한 커밋이 없었고, 주로 템플릿 개발에 초점을 맞추고 있었습니다. 따라서 이 레포를 템플릿 레포로 활용하기로 결정했습니다.

  2. 템플릿 레포로의 코드 통합

    앞서 언급했듯이, 기존 통합 개발 레포에는 추가적인 업데이트 사항들이 있었고, 이러한 업데이트들이 초기 개발/테스트용 레포(이하 템플릿 레포)에는 반영되지 않은 상태였습니다. 따라서 템플릿 분리를 완료하기 위해서는 통합 개발 레포에서 추가된 업데이트 사항들을 템플릿 레포로 가져와야 했습니다.

    이를 위해 Git의 cherry-pick 명령어를 사용하여 필요한 커밋만을 선별적으로 가져왔습니다. 먼저 통합 개발 레포를 원격 저장소로 추가한 후, 해당 커밋들을 가져왔습니다.

    # 통합 개발 레포를 원격 저장소로 추가
git remote add upstream <통합 개발 레포 주소>
git fetch upstream

# 필요한 커밋을 cherry-pick으로 가져오기
git cherry-pick <커밋 해시>

    위와 같은 방법으로 통합 개발 레포에서 템플릿과 관련된 커밋들을 선택적으로 가져와 템플릿 레포에 적용하였습니다. 이를 통해 일차적으로 템플릿 분리를 완료할 수 있었습니다.

    참고: 저는 작업 편의를 위해 Git GUI 클라이언트인 Fork를 사용했습니다. Fork는 시각적인 인터페이스를 제공하여 복잡한 Git 작업을 쉽게 수행할 수 있게 도와줍니다. 하지만 본 글에서는 이해를 돕기 위해 명령어 기반으로 설명하겠습니다.

  3. 사용자 포스트 제거

    지금의 템플릿 레포에는 contents 폴더가 존재했습니다. 이 폴더는 사용자 포스트를 관리하는 역할을 합니다.

    하지만 템플릿 레포에는 특정 사용자의 포스트나 테스트용 콘텐츠가 포함되어 있지 않아야 합니다. 따라서 기존에 테스트를 위해 작성했던 포스트나 불필요한 콘텐츠를 정리하였습니다.

  4. 분리 후 정상 동작 여부 테스트

    분리된 템플릿 레포의 정상 동작 여부를 확인하기 위해 로컬 환경에서 애플리케이션을 실행했습니다.

    npm install
npm run start

    그러나 테스트 과정에서 예상치 못한 문제가 발생했습니다. 코드는 이전과 동일한데, 기존 레포에서는 보지 못했던 오류가 발생한 것입니다.

    • GraphQL 에러: Field “thumbnail” must not have a selection since type “String” has no subfields

    이 에러의 해결 과정은 비교적 간단했지만, 문제의 원인과 해결 방법에 대한 상세한 설명은 별도의 포스트로 작성하여 링크로 연결하겠습니다.

    이 문제를 해결함으로써 템플릿 코드의 1차 분리 작업을 완료할 수 있었습니다.

마무리

이러한 과정을 거쳐 1차적인 템플릿 분리 작업을 성공적으로 마무리할 수 있었습니다. 다음 글에서는 템플릿을 서브모듈로 불러오고 포스트 레포에 배포되도록 설정하기에 대해 다뤄보겠습니다.
